🐍 Flask 后端

⬇️ SQL ⬆️

🎯 这个方案的特点

✅ 优势

  • 部署简单：pip install flask → python app.py 即可
  • 数据健壮：SQLite 事务写入，断电不丢数据
  • 零依赖前端：纯 HTML/CSS/JS，无 npm/node_modules
  • 美观现代：响应式设计，移动端也可填写
  • 单文件数据库：survey.db 可直接拷贝备份

⚠️ 需要注意

  • Flask 内置服务器适合内网小规模使用
  • 需确保部署机器的 Python 环境可用
  • 并发写入 SQLite 有锁限制（但11个团队串行填写完全够用）
